## [v0.0.3] - 2024-01-15

### Добавлено
- Поддержка кавычек для значений в settings.ini
- Обработка специальных символов в паролях (%, $, {, } и др.)
- Отключение интерполяции ConfigParser для корректной работы с %
- Улучшенный парсинг списков с поддержкой пробелов в путях
- Подробная документация по использованию кавычек

### Исправлено
- Критическая ошибка: `'%' must be followed by '%' or '(', found: '%'`
- Проблемы с паролями, содержащими специальные символы
- Обработка путей с пробелами в названиях папок

### Изменено
- Обновлен формат settings.ini с примерами в кавычках
- Дополнен README.md разделом о поддержке специальных символов

## [v0.0.2] - 2024-01-10

### Добавлено
- Вынесение настроек в отдельный файл settings.ini
- Поддержка резервного копирования с возможностью отключения
- Копирование файлов от имени другого пользователя
- Подтверждение опасных операций
- Автоматическая проверка зависимостей
- Обработка ошибок аутентификации MySQL
- Поддержка auth_plugin для старых версий MySQL

### Исправлено
- Ошибка подключения к MySQL: `'cryptography' package is required`
- Проблемы с правами доступа при копировании файлов
- Улучшена обработка ошибок для всех операций

### Изменено
- Реструктуризирован код для лучшей читаемости
- Добавлен файл requirements.txt для зависимостей
- Обновлена документация с решениями распространенных проблем

## [v0.0.1] - 2024-01-05

### Добавлено
- Базовый функционал очистки таблиц MySQL:
  - Режим `truncate` - очистка указанных таблиц
  - Режим `drop_list` - удаление указанных таблиц  
  - Режим `drop_all` - удаление всех таблиц в базе
- Очистка содержимого папок
- Копирование файлов после очистки
- Резервное копирование перед выполнением операций
- Базовая обработка ошибок
- Подтверждение деструктивных операций

### Технические особенности
- Написано на Python 3
- Использует PyMySQL для работы с MySQL
- Поддержка Linux систем
- Скрипт с настройками в переменных в начале файла

## Типы изменений

- **Добавлено** для новых функций.
- **Изменено** для изменений в существующем функционале.
- **Устарело** для функций, которые скоро будут удалены.
- **Удалено** для удаленных функций.
- **Исправлено** для исправления ошибок.
- **Безопасность** в случае уязвимостей.
